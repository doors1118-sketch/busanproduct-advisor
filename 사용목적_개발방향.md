# 지역상품 구매지원 지능형 업무 매뉴얼 — 사용 목적 및 개발 방향

## 1. 왜 만드는가 (문제 인식)

### 현장의 문제

| 문제 | 상세 |
|------|------|
| **법적 불안감** | 지역업체와 계약하려 해도, 감사·위법성 우려로 계약 담당자가 망설임 |
| **정보 분산** | 근거 법령이 지방계약법·시행령·행안부 예규·조달청 고시·부산시 조례 등에 흩어져 있음 |
| **검색 시간** | 법령 검색·해석에 건당 약 30분 소요 → 업무 부담 |
| **낮은 수주율** | 국가공공기관의 부산 지역업체 수주율 31.6% (본청 70.6% 대비 절반 이하) |

### 핵심 목적

> **계약 담당자가 지역업체와 계약할 때 느끼는 심리적 장벽을 제거하고, 법적 근거를 즉시 제시하여 지역 조달 수주율을 제고한다.**

담당자가 "이렇게 계약해도 문제없는 건가?" 하는 순간에, **법령 근거 + 판례 + 내부 지침**을 즉시 보여주는 것이 이 시스템의 존재 이유.

```
              ┌──────────▼──┐  ┌───────▼───────┐  ┌──▼──────────┐
              │ 🌐 웹앱      │  │ 💬 카카오톡    │  │ 📱 텔레그램  │

              │ +업체검색    │  │ +업체 추천    │  │ +업체 추천   │
```

| 채널 | 형태 | 제공 기능 |

| **카카오톡** | 챗봇 (프롬프트) | 기능 2(법적 해석) + 기능 3(업체 추천) 간이 응답 |
| **텔레그램** | 챗봇 (프롬프트) | 기능 2(법적 해석) + 기능 3(업체 추천) 간이 응답 |

> **웹앱**: 정밀 검색·데이터 시각화·업체 상세 정보 등 풀 기능  
> **메신저**: 이동 중 간이 조회 — "3억 공사 지역제한 가능?" → 즉시 답변

### 기능 1: 지역업체 활용 계약절차 안내 📋

> "이 사업, 지역업체를 활용하려면 어떤 절차로 가야 하나요?"

일반적인 계약절차가 아니라, **각 단계에서 지역업체를 최대한 활용할 수 있는 방법**을 안내.

| 세부 기능 | 설명 |
|------|------|
| 지역 친화 계약방법 추천 | 금액·목적물 입력 → 지역제한 경쟁, 지역 수의계약 등 **지역업체 우선 경로** 제시 |
| 절차별 지역 배려 포인트 | 8단계 프로세스마다 **"이 단계에서 지역업체를 위해 할 수 있는 것"** 표시 |
| 지역제한 가능 기준표 | 계약유형·금액별 지역제한 적용 가능 여부 한눈에 확인 |
| 공고문 작성 가이드 | 지역제한 입찰공고 시 필수 기재사항, 공동수급체 참여비율 명시 방법 |

### 기능 2: 지역업체 보호제도 법적 해석 ⚖️

> "이 계약에 지역제한을 걸 수 있나요? 법적 근거가 뭐죠?"

**핵심 기능**. 지역업체 배려 계약이 합법적인지, 근거 조항은 무엇인지 즉시 판단.

| 세부 기능 | 설명 |
|------|------|
| 지역제한 가능 여부 판단 | 금액·계약유형 입력 → 시행령 제13조 제1항 제6호 등 근거 제시 |
| 수의계약 지역업체 활용 | 소기업·소상공인 수의계약 가능 요건 (시행령 제25조) |
| 공동수급체 참여비율 | 지역업체 최소 참여비율 (시행령 제88조 제6항) |
| MAS 2단계 지역제한 | 다수공급자계약 추가경쟁 시 지역제한 적용 방법 |
| 판례·질의회신 검색 | 유사 사례의 감사 판단·법원 판결 즉시 조회 |
| AI 인용 검증 | 챗봇이 제시한 법 조항이 실존하는지 환각 방지 검증 |

답변 형식: **[결론] → [법적 근거] → [상세 해석] → [실무 적용] → [주의사항]**

### 기능 3: 지역업체 추천 🏢

> "이 물품을 납품할 수 있는 부산 소재 업체가 있나요?"

법적으로 지역제한이 가능하다고 판단된 후, **실제로 계약 가능한 지역업체를 매칭**.

| 세부 기능 | 설명 |
|------|------|
| 업체 검색 | 기존 모니터링 시스템 20,000+ 부산 소재 업체 DB 활용 |
| 면허·업종 필터링 | 18,678개 면허 업종 기준 매칭 |
| 품명 기반 추천 | 20,345개 대표 품명으로 납품 가능 업체 즉시 조회 |
| 계약 실적 조회 | 해당 업체의 과거 수주 실적·이행 이력 확인 |

> 기능 1(절차) → 기능 2(법적 해석) → 기능 3(업체 추천) 순서로 **자연스러운 업무 흐름**을 따라간다.

## 4. 어떻게 만드는가 (기술 방향)

### 시스템 동작 흐름

```
담당자 질문: "2억 물품, 부산 업체한테 수의계약 가능해?"
        │
        ▼
  ❶ Korean Law MCP (법령 검색)
     → 시행령 제25조 원문 찾기
     → 관련 판례 3건 찾기
        │
        ▼
  ❷ 자체 RAG (내규 검색)
     → 부산시 계약 지침 관련 조항 찾기
        │
        ▼
  ❸ Gemini API (해석 생성)
     → 찾은 법령 + 내규를 전달
     → AI가 종합 해석 답변 작성
        │
        ▼
  ❹ 답변 전달
     → 웹앱 / 카카오톡 / 텔레그램
```

> **Korean Law MCP** = 법령 원문을 찾아오는 검색 엔진  
> **자체 RAG** = 기관 내규를 찾아오는 검색 엔진  
> **Gemini** = 찾아온 원문을 읽고 해석해주는 AI 두뇌

### Gemini API 연결 방식

| 항목 | 내용 |
|------|------|
| Gemini Ultra 구독 (현재) | 본인이 브라우저에서 직접 대화할 때 사용 |
| **Gemini API (개발용)** | 우리 프로그램이 코드로 Gemini를 호출 — **별도 발급** |
| API 키 발급 | [Google AI Studio](https://aistudio.google.com)에서 무료 발급 |
| 호출 방식 | Python 코드 4줄로 질문 → 답변 수신 |

> 울트라 구독과 API는 **별개**. 웹앱 서비스는 API를 사용하며 사용량 기반 과금.

### 법령 데이터 활용 도구

| 도구 | 역할 | 커버 범위 |
|------|------|------|
| **Korean Law MCP** | 실시간 법령·판례 검색 | 국가법령 (L1~L3) |
| **자체 RAG** (ChromaDB) | 기관 내규·지침 검색 | 내부 문서 (L4~L6) |
| **legalize-kr** | 법령 변천사 추적 | 개정 이력 분석 |

### 데이터 계층 구조

```
L1  지방계약법·시행령·시행규칙    ← Korean Law MCP 자동
L2  행안부 계약예규              ← Korean Law MCP 자동
L3  조달청 입찰 조건·고시         ← Korean Law MCP 자동
L4  부산시 자체 계약 지침         ← 자체 RAG (관리자 업로드)
L5  자치구군·출자출연기관 내규     ← 자체 RAG (관리자 업로드)
L6  질의회신·감사원 결정례         ← 자체 RAG (수동 수집)
```

### 인프라 구성

```
NCP 서버 (기존 1대 활용)
├── 포트 A: 모니터링 시스템 (busanproduct.co.kr) ← 기존
├── 포트 B: 지역상생 어드바이저 웹앱             ← 신규
├── ChromaDB: 자체 RAG (서버 내 로컬 저장)       ← 신규
├── Korean Law MCP: 백그라운드 프로세스           ← 신규
└── Webhook: 카카오톡/텔레그램 챗봇 연동          ← 신규
```

> **1단계**: 기존 서버에서 시작 (추가 비용 0원)  
> **2단계**: 사용자 증가 시 인스턴스 분리 검토

> 프롬프트 엔지니어링(질문을 잘 쓰자)의 상위 개념.

### ❶ 정체성 (Identity)

```
당신은 "지역상생 조달 어드바이저"입니다.
부산광역시 공공조달 계약 법무 전문 자문관으로서,
계약 담당자가 지역업체와 계약할 때 발생하는 법적 고민을 해결합니다.

핵심 가치:
- 지역업체 활용이 가능한 방향을 우선 안내합니다
- 모든 답변에 법령 조항 번호를 인용합니다
- 감사 리스크가 있으면 반드시 경고합니다
- 확실하지 않으면 솔직하게 "확인이 필요합니다"라고 말합니다
```

### ❷ 도구 흐름 (Tool Orchestration)

질문이 들어오면 AI가 **반드시 이 순서로 행동**하도록 제어:

```
사용자 질문 입력
    │
    ▼
[1단계] 질문 분류
    ├─ 계약절차 질문  → 기능 1 흐름
    ├─ 법적 해석 질문  → 기능 2 흐름
    ├─ 업체 추천 질문  → 기능 3 흐름
    └─ 복합 질문      → 기능 2 → 3 순차 실행
    │
    ▼
[2단계] 데이터 수집 (반드시 실행)
    ├─ Korean Law MCP: search_law → 관련 법령 원문
    ├─ Korean Law MCP: search_interpretations → 해석례
    ├─ Korean Law MCP: search_decisions → 판례 (필요시)
    └─ 자체 RAG: ChromaDB → 기관 내규 (해당시)
    │
    ▼
[3단계] Gemini 해석 생성
    │   수집된 법령 원문 + 내규를 첨부하여 답변 생성
    ▼
[4단계] 인용 검증
    │   verify_citations → 답변에 인용된 조문이 실존하는지 확인
    ▼
[5단계] 답변 전달
```

> **핵심 규칙**: 2단계(데이터 수집) 없이 3단계(해석)로 건너뛰기 **금지**.
> AI가 기억에서 답변하는 것을 원천 차단.

```
⚖️ 본 답변은 참고용이며 법적 효력이 없습니다.
```

### ❹ 행동 제약 (Behavioral Constraints)

| 규칙 | 설명 |
|------|------|
| **법령 근거 없는 해석 금지** | 조항 번호를 인용할 수 없으면 답변하지 않음 |
| **추측 답변 금지** | "~인 것 같습니다" 대신 "확인이 필요합니다" |
| **역할 이탈 금지** | 계약·조달 외 질문에는 "전문 범위를 벗어납니다" |
| **면책 고지 필수** | 모든 답변 하단에 "참고용이며 법적 효력 없음" 표시 |
| **감사 경고 의무** | 위법 소지가 있는 계약 방법은 반드시 리스크 고지 |
| **개인정보 차단** | 업체 대표자 주민번호 등 민감정보 노출 차단 |

### ❺ 분기 판단 (Decision Tree)

**기능 1: 지역업체 활용 계약절차 판단 로직**

```
입력: 금액, 계약유형 (공사/물품/용역)
    │
    ├─ 공사 + 종합 ≤ 3억  → 지명경쟁 가능 (시행령 제22조)
    ├─ 공사 + 전문 ≤ 1억  → 지명경쟁 가능
    ├─ 물품/용역 ≤ 2천만  → 수의계약 가능 (제25조 제5호 나목)
    ├─ 물품/용역 ≤ 1억 + 소기업  → 수의계약 가능 (제25조 제5호 라목)
    ├─ 행안부령 기준 미만  → 지역제한 경쟁 가능 (제13조 제6호)
    └─ 기준 초과           → 일반경쟁, 공동수급체 참여비율 활용
```

**기능 2: 지역제한 가능 여부 판단 로직**

```
입력: 금액, 계약유형, 계약방식
    │
    ├─ 지방계약법 적용 대상? ──[No]──→ 국가계약법 확인 안내
    │   [Yes]
    ├─ 추정가격 행안부령 기준 미만? ──[Yes]──→ ✅ 지역제한 가능
    │   [No]                                     (제13조 제6호)
    ├─ 공동수급체 방식 가능? ──[Yes]──→ ✅ 지역업체 참여비율 설정
    │   [No]                              (제88조 제6항)
    └─ MAS 2단계 경쟁? ──[Yes]──→ ✅ 추가경쟁 시 지역제한 검토
        [No]
        → ⚠️ 지역제한 어려움, 대안 안내
```

### 하네스 적용 전후 비교

**하네스 없이:**
```
Q: 2억 물품 수의계약 가능해?
A: 네, 수의계약이 가능할 수 있습니다. 관련 법령을 확인하세요.
   → 근거 없음, 실무 도움 안 됨
```

**하네스 적용 후:**
```
Q: 2억 물품 수의계약 가능해?

📌 결론: 일반적인 수의계약은 불가능합니다. 단, 소기업·소상공인 대상 시 가능합니다.

📜 법적 근거:
  • 시행령 제25조 제1항 제5호 나목: 물품 수의계약 한도 2천만원 이하
  • 시행령 제25조 제1항 제5호 라목: 소기업·소상공인 대상 시 1억원 이하
  • 2억원은 두 기준 모두 초과 → 수의계약 불가

💼 실무 적용:
  • 대안 1: 지역제한 제한경쟁입찰 (시행령 제13조 제6호)
  • 대안 2: 공동수급체 구성 시 지역업체 참여비율 설정

⚠️ 주의사항:
  • 소기업 기준은 업종별 매출액 기준 상이 (중소기업기본법 제2조)
  • 분할발주로 수의계약 기준 미만으로 나누는 것은 위법 (감사 적발 사례 있음)

⚖️ 본 답변은 참고용이며 법적 효력이 없습니다.
```
