# 부산 공공조달 AI 챗봇 — 내부 파일럿 실행 체크리스트

본 문서는 사무실 내부 인원을 대상으로 한 1일 파일럿 테스트를 수행하기 전, 수행 중, 그리고 종료 후에 점검해야 할 사항들을 정리한 체크리스트입니다. 실제 서비스 실행 및 ACG 변경은 **관리자의 최종 승인**이 이루어진 후에만 진행해야 합니다.

> **Production deployment: HOLD**
> 본 테스트는 정식 운영 환경 배포가 아니며, 기존 운영 서비스 아키텍처에 영향을 주지 않습니다.

## 1. 파일럿 테스트 요약
- **접속 URL:** `http://49.50.133.160:8001/ui/`
- **대상 및 기간:** 사무실 제한 인원 대상, 하루 동안 진행
- **접근 제어 (ACG):** 가급적 사무실 공인 IP 대역 허용 권장 (0.0.0.0/0 절대 금지)
- **인증 제어:** Basic Auth 적용 (파일럿 전용 ID/PW)

## 2. 파일럿 실행 전 사용자 결정 항목
다음 항목들의 구체적인 값은 사전에 내부적으로 결정하되, 어떠한 설정값도 문서나 코드에 직접 하드코딩하거나 기록하지 않습니다.
- [ ] 파일럿용 ID 및 임시 비밀번호 결정
- [ ] 테스트 시작 및 종료 일시 결정
- [ ] 테스트 대상 직원 및 접근할 사무실 공인 IP 대역 식별

## 3. 백엔드 실행 방식 (택 1)
### A안: 임시 스크립트 기반 (단기 테스트용)
- `nohup` 또는 `tmux`를 활용하여 `python3 -m uvicorn app.api_server:app --host 0.0.0.0 --port 8001` 실행
- 장점: 즉시 구동 가능하고 종료가 간편함
- 단점: 예기치 않은 프로세스 종료 시 자동 복구 불가능

### B안: 임시 systemd 서비스 (권장 - 1일 지속 테스트용)
- 하루 동안 다수의 직원이 테스트할 경우 더 안정적인 운영 방식
- `busan-advisor-api-pilot.service` 와 같은 별도 서비스 파일 생성 후 백그라운드 구동 (실행은 사용자 명시적 승인 후 진행)

## 4. 파일럿 오픈 전 최종 점검 (Smoke Test)
- [ ] `GET /ui/` (Basic Auth 없이 접근 시 401 Unauthorized 반환 확인)
- [ ] `GET /ui/` (올바른 Auth 정보로 접근 시 프런트엔드 화면 정상 로드 확인)
- [ ] `GET /rag/status` (올바른 Auth 정보로 접근 시 200 OK 및 내부 컬렉션 상태 반환)
- [ ] `POST /chat` (올바른 Auth 정보로 벤치마크 질문 수행, 200 OK 반환 확인)

## 5. 파일럿 종료 후 필수 조치
테스트가 예정된 종료 시간에 도달하면 반드시 아래 절차를 수행하여 보안 상태를 복구합니다.
- [ ] FastAPI 프로세스 중지 (A안의 경우 `kill`, B안의 경우 `systemctl stop`)
- [ ] NCP ACG의 `TCP 8001` 허용 규칙 비활성화 또는 삭제
- [ ] 발급했던 파일럿용 임시 ID/PW 폐기 (재사용 금지)
- [ ] 테스트 참여 직원들로부터 전달된 피드백 문서 수합 및 검토

---
**주의:** 
실행 시 아래의 명령을 활용하되, 실제 ID와 PW는 로컬 환경변수에만 설정하며 출력 및 로그 기록을 금지합니다.
```bash
export PILOT_AUTH_ENABLED=true
export PILOT_AUTH_USER=<set_in_shell>
export PILOT_AUTH_PASSWORD=<set_in_shell>
python3 -m uvicorn app.api_server:app --host 0.0.0.0 --port 8001
```
