# 부산 공공조달 AI 챗봇 — Operations Runbook

본 문서는 공공조달 AI 챗봇 시스템의 원활한 운영 및 장애 대응을 위한 가이드라인입니다.

## 1. 시스템 실행 및 확인

### API 서버 구동
- **위치**: `/opt/busan` (운영 디렉터리)
- **명령**:
  ※ 아래 명령은 운영자 승인 후에만 실행한다. 현재 단계에서는 실행 금지.
  ```bash
  cd /opt/busan
  # 테스트 또는 임시 실행 시
  python -m uvicorn app.api_server:app --host 127.0.0.1 --port 8001
  # (실 운영 시에는 systemd 혹은 pm2 등을 사용하여 백그라운드 구동 권장)
  ```

### Streamlit UI 실행 및 점검
- **UI 포트**: 8502 (기본)
- **접근 확인**: 브라우저를 통해 `http://서버IP:8502` 접속 또는 `curl -I http://127.0.0.1:8502`로 상태 점검.

## 2. 모니터링 및 상태 확인

### 로그 확인
시스템 로그의 확인 위치는 서비스 관리 데몬(systemd/pm2)의 표준 설정에 의존합니다.
※ 아래 명령은 운영자 승인 후에만 실행한다. 현재 단계에서는 실행 금지.
- systemd 사용 시: `journalctl -u law-chatbot.service -f`
- uvicorn 직접 구동 시: 백그라운드 리다이렉션 로그 (예: `uvicorn.log`) 참고.

### RAG 데이터 상태 점검
정상적으로 벡터 데이터베이스(ChromaDB)가 구성되어 있는지 확인하기 위해 아래 API를 활용합니다.
※ 아래 명령은 운영자 승인 후에만 실행한다. 현재 단계에서는 실행 금지.
```bash
curl -s http://127.0.0.1:8001/rag/status
```
반환된 JSON 응답 내 `laws`, `manuals`, `innovation` 필드가 `SUCCESS`인지 확인합니다.

## 3. 장애 상황별 대응 절차

### API 503 / 429 에러 발생 시
- **증상**: 외부 Gemini 모델 또는 NCP API의 Rate Limit 초과 및 과부하.
- **대응**:
  시스템은 Fail-closed 메커니즘에 의해 사용자에게 "API 사용량 한도 초과 또는 서버 지연"을 렌더링하고 `candidate_table_source="none"` 상태로 Fallback 합니다. 운영자는 일정 시간 이후 트래픽이 정상 궤도로 돌아왔는지 로그 기반으로 확인해야 합니다. 임의 프로세스 재기동은 불필요합니다.

### `result_status=DEGRADED` 발생 시
- **증상**: 전체 시스템 중 일부 데이터 소스(RAG) 연결 지연 또는 외부 API 지연.
- **대응**:
  API 자체는 죽지 않고 응답을 생성하나 신뢰성이 일부 저하되었음을 의미합니다. RAG 경로 누락이나 Chroma DB 파일 권한 문제를 점검하십시오.

### `false_pass_detected` 발생 시
- **증상**: 법적 근거가 부재함에도 비정상적인 결론이 `PASS`로 인가되는 현상.
- **대응**:
  안전 검증 모듈 또는 LLM 프롬프트가 우회당한 보안 이벤트입니다. 즉각 `/chat` 사용을 잠정 중지시키거나 `production_deployment`를 `HOLD`로 강제 복귀시킨 뒤, 원인을 패치하기 전까지 서비스 재개를 보류하십시오.

## 4. 운영 환경 제약 (절대 금지 항목)

- **수동 코드 패치 금지**: `/opt/busan` 디렉터리 내의 파이썬 코드 및 설정을 관리자 승인 없이 직접 수정하는 행위 금지.
- **서비스 임의 기동 중단 금지**: 관리자의 사전 고지 및 승인 없이 `systemctl restart`, `pm2 restart`를 실행하여 캐시 데이터를 파괴하거나 서비스를 중단시키는 행위 금지.
- **DB 원본 파괴 금지**: RAG 데이터가 저장된 운영 DB 및 `.chroma` 디렉터리 삭제 절대 금지.
- **ChromaDB Symlink 우회 금지**: 시스템 무결성에 혼란을 줄 수 있는 심볼릭 링크 처리 금지.

## 5. 비상 연락처 및 메모
