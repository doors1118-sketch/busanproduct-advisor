# 지역업체 검색 파이프라인 (API 및 적재 데이터 활용 구조)

지역상품 구매확대 챗봇의 프롬프트에서 `search_local_company...` 계열의 도구를 호출할 때, 백엔드(`app/company_api.py`)에서 수행되는 외부 API 및 적재 데이터 활용 파이프라인은 다음과 같은 4단계로 구성됩니다.

## 1. 외부 API 활용 ①: 지역 조달업체 1차 검색 (`busanproduct.co.kr`)
프롬프트 지시사항에 따라 제미나이가 구매 의도를 파악하고 도구를 호출하면, 가장 먼저 **부산상품 통합 플랫폼 외부 API**로 HTTP GET 요청을 보냅니다.
*   **역할:** "LED" 등 질의와 관련된 조달청 등록 부산 지역업체의 목록, 주소, 업종, 사업자번호 등 기초 데이터를 조회합니다.

## 2. 외부 API 활용 ②: 국세청 휴폐업 실시간 검증 (`api.odcloud.kr`)
1차로 검색된 업체들이 현재 정상 영업 중인지 확인하기 위해 **국세청 사업자등록 상태조회 공공 API**를 즉시 호출합니다.
*   **역할:** 사업자번호를 최대 100건씩 묶어 국세청 API에 전송한 뒤, 상태코드 `02(휴업)`나 `03(폐업)`인 업체를 리스트에서 완전히 제거합니다.
*   **효과:** 제미나이가 폐업한 업체를 추천하는 환각(Hallucination) 및 오류를 원천 차단합니다. (속도 저하 방지를 위해 24시간 단위의 로컬 캐싱 적용)

## 3. 내부 적재 데이터(DB) 활용: 정책기업 맵핑 (`policy_companies.py`)
국세청 검증을 통과한 '계속사업자'들의 사업자번호를, 서버에 **로컬 DB(엑셀 등)로 적재되어 있는 우대 기업 명단**과 대조(Enrichment)합니다.
*   **적재 데이터 출처:** `여성기업 조달업체 내역.xlsx`, `사회적기업 내역.xlsx` 등 내부 정적 파일
*   **역할:** 해당 업체가 우대 혜택을 받을 수 있는 조건인지 식별하여, 결과 데이터에 `<여성기업>`, `<사회적기업>` 등의 **정책기업 태그**를 추가로 부여합니다.

## 4. 프롬프트 통제: 개인정보 차단 및 포맷팅 (`format_company_results`)
위 3단계를 거친 최종 결과물은 제미나이에게 전달되기 전, 프롬프트의 **"개인정보 노출 금지"** 규칙을 준수하기 위해 한 번 더 정제됩니다.
*   **역할:** 사업자등록번호나 대표자 전화번호 등 민감한 개인정보를 필터링합니다.
*   **최종 전달 형태:** 오직 **[업체명, 소재지, 대표품목, <정책기업 태그>]** 형태의 안전하고 구조화된 텍스트만 LLM의 Context에 주입됩니다.

---
**[핵심 요약]**
챗봇 프롬프트에서의 단순한 도구 호출 한 번은, 백엔드에서 **① 조달업체 API(1차 검색) + ② 국세청 API(팩트체크) + ③ 로컬 엑셀 DB(혜택 정보 결합)**를 순식간에 융합하여 제미나이에게 완벽한 실무 데이터를 제공하는 파이프라인으로 작동합니다.
