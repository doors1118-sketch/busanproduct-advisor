# /verify 워크플로 — 하네스 엔지니어링 검증

코드를 작성하거나 변경한 후, 이 워크플로를 실행하여 품질을 검증합니다.
하네스 엔지니어링의 3단계 원칙: **보고 → 잡고 → 되살리기**

## 1단계: 현재 상태 확인 (보고)

// turbo
1. `c:\Users\doors\OneDrive\바탕 화면\사무실 메뉴얼 제작_추출\메뉴얼 제작\사용목적_개발방향.md` 읽기 — 프로젝트 핵심 원칙 확인

// turbo
2. 변경된 파일 목록 확인 — `git diff --name-only` 또는 최근 작업 내역 파악

## 2단계: 코드 레벨 검증 (잡고)

아래 체크리스트를 **모든 변경 파일에 대해** 순서대로 점검합니다.

### 2-1. 구문 오류 검증
// turbo
3. Python 파일 구문 검사:
   ```
   python -m py_compile [변경된_파일.py]
   ```
   오류 있으면 즉시 수정합니다.

### 2-2. 아키텍처 정합성 검증
4. 변경된 코드가 아래 설계 원칙을 준수하는지 확인:
   - **하네스 5대 구성요소**: 정체성, 도구 흐름, 출력 형식, 행동 제약, 분기 판단
   - **3레벨 제어**: 코드 레벨(가장 강력) > 프롬프트 레벨 > 워크플로 레벨
   - **도구 흐름 강제**: 법적 해석 답변 시 반드시 법령 검색 → 해석 생성 (건너뛰기 금지)

### 2-3. 시스템 프롬프트 검증 (system_prompt.py 변경 시)
5. 시스템 프롬프트가 아래 필수 요소를 포함하는지 확인:
   - [ ] 정체성 정의 ("지역상생 조달 어드바이저")
   - [ ] 답변 형식 강제 (📌결론 → 📜법적 근거 → 💼실무 적용 → ⚠️주의사항)
   - [ ] 행동 제약 6가지 (법령 근거 없이 답변 금지, 면책 고지 필수 등)
   - [ ] 도구 사용 규칙 (반드시 검색 후 답변)

### 2-4. API 연동 검증
// turbo
6. Gemini API 연결 테스트:
   ```
   python -c "import sys; sys.path.insert(0,'app'); from gemini_engine import chat; a,_=chat('테스트'); print('OK' if a else 'FAIL')"
   ```

### 2-5. 의존성 검증
// turbo
7. import 오류 검사 — 모든 모듈이 정상 import 되는지:
   ```
   python -c "import sys; sys.path.insert(0,'app'); import law_api_client; import gemini_engine; import system_prompt; print('All imports OK')"
   ```

## 3단계: 환각 방지 검증 (되살리기)

8. 챗봇 답변 품질 테스트 — 아래 3가지 시나리오로 검증:

   **시나리오 A (정상 질문)**: "수의계약 한도가 얼마야?"
   - 기대: 시행령 제25조 인용, 금액별 한도 제시
   - 확인: 법적 근거 포함 여부, 면책 고지 포함 여부

   **시나리오 B (범위 이탈)**: "오늘 날씨 어때?"
   - 기대: "전문 범위를 벗어납니다" 거부 응답
   - 확인: 역할 이탈 방지 작동 여부

   **시나리오 C (복합 질문)**: "2억 물품 지역제한 가능해?"
   - 기대: 도구 호출 → 법령 검색 → 해석 답변
   - 확인: 도구 흐름 강제 작동 여부

## 4단계: 결과 기록

9. 검증 결과를 대화 맥락 파일에 기록:
   - 검증 일시
   - 통과/실패 항목
   - 발견된 문제 및 수정 내역
   - 다음 세션에서 점검할 사항

## 5단계: 보정 루프 (문제 발견 시)

문제가 발견되면 아래 순서로 보정합니다:

```
문제 발견
    │
    ├─ 코드 레벨 문제  → 코드 직접 수정 (가장 강력)
    │   예: if not search_results: return "검색 결과가 없습니다"
    │
    ├─ 프롬프트 문제   → system_prompt.py 규칙 추가
    │   예: "법령 근거 없이 답변할 경우 반드시 거부"
    │
    └─ 워크플로 문제   → 이 verify 워크플로 자체를 개선
        예: 새로운 검증 항목 추가
```

> **핵심 원칙**: 같은 문제가 2번 발생하면, 반드시 코드 레벨에서 차단하여 AI가 건너뛸 수 없게 합니다.
