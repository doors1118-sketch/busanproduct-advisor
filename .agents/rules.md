# 프로젝트 규칙 — 지역상생 조달 어드바이저

## 자동 검증 규칙 (하네스 엔지니어링)

### 코드 변경 후 자동 검증
Python 파일(.py)을 생성하거나 수정한 후, 반드시 다음을 자동 실행합니다:
1. `python -m py_compile [파일]` — 구문 오류 검사
2. 해당 파일의 import가 정상 동작하는지 확인
3. 오류 발견 시 즉시 수정 후 재검증

### 시스템 프롬프트 변경 시 필수 확인
`system_prompt.py`를 수정할 때는 아래 5가지가 반드시 포함되어 있는지 확인합니다:
- 정체성 정의 ("지역상생 조달 어드바이저")
- 답변 형식 강제 (📌결론 → 📜법적 근거 → 💼실무 적용 → ⚠️주의사항)
- 행동 제약 (법령 근거 없이 답변 금지 등 6가지)
- 도구 사용 규칙 (반드시 검색 후 답변)
- 면책 고지 필수

### 보정 원칙 (3레벨)
같은 문제가 2번 발생하면 반드시 코드 레벨에서 차단합니다:
- **코드 레벨** (가장 강력): if/assert로 강제 — AI가 건너뛸 수 없음
- **프롬프트 레벨**: system_prompt.py에 규칙 추가
- **워크플로 레벨**: .agents/workflows/ 에 절차 추가

## 세션 관리 규칙
- 세션 시작: `/start` 워크플로 실행
- 세션 종료: `/end` 워크플로 실행 (하네스 검증 포함)
- 코드 검증: `/verify` 워크플로 (수동 실행 시)

## 자동 실행(Auto-Run) 규칙
- 안전을 보장할 수 있는 파이썬 실행, 조회, 테스트, 빌드 명령어는 특수한 경우를 제외하고 반드시 `SafeToAutoRun=true`로 설정하여 사용자의 승인을 기다리지 말고 자동 실행할 것.
